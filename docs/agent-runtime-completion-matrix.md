# Agent Planner Runtime 完成度验收矩阵

更新时间：2026-07-07

目的：把用户目标“真正实现 Agent Planner 方式；每一个沟通严格匹配 Agent Planner；每一个功能都支持；分析对话历史；实现所有功能”拆成可验证 requirement，避免把局部 smoke、单个 live 请求、或静态配置误当作全局完成。

## 完成判定原则

- **不能只看一个 global admin audit**：global audit 混合历史 session、匿名请求、不同 runtime store 和 bounded event window；只能用于 operator overview。
- **conversation strict proof 必须 scoped**：必须用 tenant/workspace/session scoped audit 或 protocol strict smoke 证明当前会话有 `intent_classification` 且进入 planner-owned synthesis/tool boundary。
- **所有功能 proof 必须执行公开 surface**：以 `/healthz.supported_paths` 为契约，逐个调用 advertised path，并验证 conversation / Gateway-owned endpoint 的 runtime evidence。
- **历史分析 proof 必须区分 follow-up 与污染**：历史只可用于明确 follow-up、tool_result/evidence 轮，不得让 “谢谢/jo/ok” 或旧 workspace 触发工具。
- **chat-only upstream 永远不是 Planner**：上游只能最终措辞；拒答、串旧 session/workspace、无工具占位必须被 Gateway final-synthesis guard 接管。
- **多用户 proof 必须并发验证**：同名 session/tool/cache/memory 在不同 tenant/workspace 下不得串。
