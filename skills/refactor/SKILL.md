# Refactor 安全重构

在不改变行为的前提下改善代码结构。

## 重构流程
1. **锁定** — 确认现有测试全部通过
2. **识别** — 找到代码坏味道
3. **选择** — 确定重构手法
4. **执行** — 小步前进，每步可回退
5. **验证** — 每步后运行测试
6. **提交** — 单独 commit，不混入功能变更

## 代码坏味道
- 函数超过 50 行
- 嵌套超过 3 层
- 参数超过 4 个
- 重复代码出现在 3 处以上
- 类职责不单一
- 过深的继承链

## 重构手法
- 提取函数 / 提取类
- 内联函数
- 搬移方法 / 字段
- 用查询替换临时变量
- 引入参数对象
- 用多态替换条件分支

## 规则
- 重构和功能开发分开 commit
- 不在重构中添加新功能
- 每次只做一种重构
- 测试不通过就回退
