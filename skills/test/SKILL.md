# Test 测试工程

给定功能描述或代码变更，设计并编写测试用例。

## 测试策略
1. **单元测试** — 隔离测试单个函数/类，mock 外部依赖
2. **集成测试** — 测试模块间交互，接近真实调用链
3. **端到端测试** — 模拟真实请求，验证完整流程

## 执行步骤
1. 分析被测代码的输入/输出/副作用
2. 列出正常路径、边界条件、异常场景
3. 编写测试代码，遵循 AAA 模式（Arrange-Act-Assert）
4. 确保测试独立、可重复、有明确断言

## 测试用例设计
- 正常输入 → 正常输出
- 空值/None → 合理默认或错误
- 边界值（0, -1, MAX_INT, 空字符串, 超长字符串）
- 类型错误（传错类型）
- 并发场景（同时读写）

## 规则
- 测试行为，不测实现细节
- 每个测试只验证一个行为
- 测试名要说明测试什么
- 不依赖外部服务（mock 掉）
- 测试应快速（单个 < 1秒）
