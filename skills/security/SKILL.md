# Security 安全审查

对代码或配置执行安全审查，识别漏洞和风险。

## 审查清单
1. **输入验证** — 所有外部输入是否校验？有无注入风险？
2. **认证授权** — 权限边界是否清晰？有无越权可能？
3. **敏感数据** — API Key/密码是否加密存储？日志是否脱敏？
4. **依赖安全** — 第三方库是否有已知漏洞？
5. **配置安全** — 默认值是否安全？有无不必要的开放端口？
6. **网络安全** — 是否有 SSRF/CSRF 风险？
7. **错误处理** — 错误信息是否泄露内部细节？

## Gateway 专项
- 上游 API Key 是否明文存储或传输
- 下游认证是否可绕过
- 工具执行是否有沙箱隔离
- 文件读写是否限制在 workspace 内
- Shell 工具是否有命令注入风险
- 日志中是否记录了敏感请求体

## 规则
- 安全问题必须标注严重级别（Critical/High/Medium/Low）
- 每个问题给出具体修复建议
- 不因为"概率低"而降低严重级别
- 发现 Critical 问题立即报告
