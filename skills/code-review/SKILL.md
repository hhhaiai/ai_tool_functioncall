# Code Review 代码审查

给定代码片段或文件路径，执行结构化代码审查。

## 审查维度
1. **正确性** — 逻辑错误、边界条件、空值处理、类型错误
2. **安全性** — 注入、认证绕过、敏感信息泄露、权限越界
3. **性能** — O(n²) 循环、N+1 查询、内存泄漏、不必要的拷贝
4. **可维护性** — 命名清晰度、函数长度、耦合度、重复代码
5. **错误处理** — 异常是否被捕获、错误信息是否有用、降级策略

## 执行步骤
1. 先通读文件，理解整体职责
2. 逐函数/逐类审查，记录问题
3. 按严重性分级：P0(阻断) P1(严重) P2(建议) P3(风格)
4. 输出结构化报告，每个问题引用具体文件和行号

## 输出格式
```
[P级别] 文件:行号 — 问题描述
  原因: 为什么这是问题
  修复: 具体修改建议
```

## 规则
- 只报告代码中确认的问题，不猜测
- 不改变功能的前提下提出改进建议
- 安全问题一律 P0/P1
