# Performance 性能优化

分析代码或系统瓶颈，给出优化建议。

## 分析步骤
1. **度量** — 先测量，用数据说话，不猜测
2. **定位** — 找到真正的热点（profiling）
3. **优化** — 针对瓶颈做最小改动
4. **验证** — 重新度量，确认提升且无副作用

## 常见优化方向
- **算法** — O(n²) → O(n log n)，减少不必要的遍历
- **IO** — 批量操作、异步化、缓存命中率
- **内存** — 减少分配、对象复用、及时释放
- **并发** — 连接池、限流、无锁数据结构
- **网络** — 压缩、keep-alive、减少往返

## Gateway 专项
- 上游请求超时优化（连接池复用、DNS 缓存）
- 流式响应延迟（缓冲策略、flush 时机）
- 上下文压缩效率（摘要质量 vs 压缩率）
- 缓存命中率（语义相似度阈值调整）
- 并发请求调度（队列深度、worker 数量）

## 规则
- 没有度量数据不提优化建议
- 优化后必须有 before/after 对比
- 不做 premature optimization
- 优先优化热路径
