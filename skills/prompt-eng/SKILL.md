# Prompt Engineering 提示工程

为上游 LLM 设计高效的提示词，提升输出质量。

## 核心技巧
1. **角色设定** — 明确专业角色和能力边界
2. **任务分解** — 复杂任务拆成小步骤
3. **格式约束** — 指定输出格式（JSON/Markdown/表格）
4. **示例引导** — 提供输入输出示例
5. **链式推理** — 要求分步思考
6. **自我检查** — 要求验证自己的输出

## Gateway 文本工具适配
当上游不支持原生 tool call 时，需要将工具描述注入 prompt：

```
你可以使用以下工具:

## 工具名称
描述: ...
参数:
  - name (string, required): ...
  - value (string, optional): ...

要调用工具，请输出:
<tool_call>{"name":"tool_name","arguments":{...}}</tool_call>
```

## 提示词优化
- 测试不同措辞的效果
- 添加负面示例（什么是不要的）
- 用分隔符区分指令和上下文
- 控制提示词长度，避免信息过载
- 迭代优化，每次只改一个变量

## 规则
- 基于实际场景设计，不套模板
- 测试验证效果，不靠直觉
- 记录每次优化的结果
