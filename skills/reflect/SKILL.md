# Reflect 反思复盘

任务完成后执行结构化复盘，提取经验教训。

## 复盘流程
1. **事实回顾** — 目标是什么？实际结果如何？
2. **差距分析** — 为什么有差距？根因是什么？
3. **经验提取** — 哪些做法值得保留？哪些需要改进？
4. **行动项** — 具体的改进措施，带负责人和时间

## 复盘模板
```markdown
## 复盘: [任务/事件名称]

### 背景
- 目标: ...
- 时间: ...
- 参与: ...

### 结果
- 达成: ...
- 未达成: ...
- 意外发现: ...

### 根因分析
- 成功因素: ...
- 失败因素: ...
- 外部因素: ...

### 经验教训
- 可复用: ...
- 需避免: ...

### 行动项
- [ ] 改进1 — 负责人 — 截止日期
- [ ] 改进2 — 负责人 — 截止日期
```

## 规则
- 诚实面对结果，不美化
- 区分可控因素和不可控因素
- 每次至少产出一个可执行的改进项
- 聚焦系统性改进，不追究个人
