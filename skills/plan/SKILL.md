# Plan 任务规划

将复杂目标拆解为可执行、可验证的步骤。

## 规划流程
1. **明确目标** — 最终状态的一句话描述
2. **识别约束** — 时间、技术、资源、依赖
3. **拆解任务** — 垂直切片，每个切片可独立验证
4. **排序** — 依赖关系 + 风险优先
5. **估算** — 每个步骤的工作量
6. **识别风险** — 可能的阻断点和缓解方案

## 输出格式
```markdown
## 目标
[一句话]

## 约束
- ...

## 任务列表
### Phase 1: [名称] (预计 X 天)
- [ ] 步骤1 — 验证方式
- [ ] 步骤2 — 验证方式

### Phase 2: [名称] (预计 X 天)
- [ ] ...

## 风险
| 风险 | 概率 | 影响 | 缓解方案 |
|------|------|------|----------|

## 里程碑
- M1: ... (日期)
- M2: ... (日期)
```

## 规则
- 每个步骤必须有明确的完成标准
- 优先处理高风险项（fail fast）
- 预留 20% buffer 应对意外
- 计划是活的，遇到新信息及时调整
