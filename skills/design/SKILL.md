# Design 方案设计

给定需求或问题，输出结构化的设计方案。

## 设计流程
1. **需求分析** — 明确目标、约束、边界
2. **方案设计** — 至少考虑两种方案，对比优劣
3. **接口设计** — 定义输入/输出/错误处理
4. **数据设计** — 数据结构、存储方案、索引策略
5. **风险评估** — 识别风险并给出缓解措施

## 输出格式
```markdown
## 目标
一句话说明要解决什么问题

## 约束
- 技术约束: ...
- 资源约束: ...
- 时间约束: ...

## 方案对比
| 维度 | 方案A | 方案B |
|------|-------|-------|
| 复杂度 | ... | ... |
| 性能 | ... | ... |
| 可维护性 | ... | ... |

## 推荐方案
选择方案X，因为...

## 详细设计
### 接口
### 数据结构
### 核心流程

## 风险
| 风险 | 概率 | 影响 | 缓解 |
```

## 规则
- 设计决策必须有理由
- 明确标注被拒绝的方案及原因
- 优先简单方案，不过度设计
