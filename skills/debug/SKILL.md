# Debug 调试诊断

给定错误信息、日志或异常行为，执行系统化诊断。

## 诊断流程
1. **复现** — 确认问题可稳定复现，记录触发条件
2. **定位** — 从错误信息追溯到源头，缩小范围
3. **假设** — 列出可能的根因，按概率排序
4. **验证** — 用最小手段验证每个假设（日志/断点/测试）
5. **修复** — 最小改动修复，不引入新问题
6. **防护** — 添加测试/断言防止回归

## 常见 Gateway 问题模式
- 上游超时 → 检查 timeout 配置和网络连通性
- 协议不匹配 → 对比上游实际响应和预期格式
- 工具执行失败 → 检查工具参数 schema 和执行环境
- 流式中断 → 检查 SSE 解析和缓冲区处理
- 并发冲突 → 检查锁/共享状态/连接池
- 内存增长 → 检查缓存未清理/连接未关闭

## 规则
- 先看日志和错误信息，再看代码
- 用二分法缩小范围
- 每个假设必须有证据支撑
- 修复后必须验证：原问题不复现 + 无新问题
