# Doc 文档生成

给定代码或功能描述，生成对应的技术文档。

## 文档类型
- **README** — 项目概览、快速开始、配置说明
- **API 文档** — 接口定义、参数、返回值、示例
- **配置说明** — 配置项含义、默认值、使用场景
- **架构文档** — 系统分层、数据流、模块职责
- **变更日志** — 版本变更摘要

## 执行步骤
1. 分析代码/功能的实际行为
2. 提取关键信息：输入、输出、配置、依赖
3. 按目标文档类型组织内容
4. 添加可运行的代码示例
5. 检查准确性和完整性

## 写作规则
- 开头一句话说清楚是什么/做什么
- 代码示例必须可直接运行
- 参数表格优于长段落
- 中英文术语保持一致
- 不写废话，每句话都有信息量

## 规则
- 基于实际代码生成，不臆造功能
- 保持文档和代码同步
- 敏感信息（API Key、密码）用占位符
